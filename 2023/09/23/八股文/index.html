JVM、JRE和JDK

  • JVM是java运行时的虚拟机，不同平台都有各自的虚拟机，从而支持java的跨平台
  • JRE是虚拟机和程序所需要的核心类库（java.lang等）。想要在机器上运行java，装一个jre就行了
  • JDK是给开发人员使用的，包括了JRE。其中的开发工具有编译工具（javac.exe）、打包工具（jar.exe）等

字节码和其好处

  • 字节码是java代码编译后的文件(.class)，只面向虚拟机。
  • 好处： 解决了传统解释型语言执行效率低的 问题，同时又保留了解释型语言可移植的特点。无需重新编译就可以在不同系统运行。

访问修饰符

  • private：同一类之内可见。修饰变量、方法
  • default：同一包内可见。修饰类、接口、变量、方法
  • protected：同一包内的类和子类可见。修饰变量、方法
  • public：全局可见。修饰类、接口、变量、方法

&和&&

  • &：两种用法：逻辑与和按位与
  • &&：短路运算符，左边false就不执行右边

final关键字

在代码中不可被继承、重写、改变。但是final修饰的变量不可变的是引用，如果引用指向的地址变了，那这个变量的内容就变了

跳出多重循环

在想跳出的循环钱加上标号，里层循环使用带有标号的break，就可以跳出外层循环。

面向对象五大原则

  • 单一职责：类功能要单一，从而解耦合
  • 不可修改：可以拓展，不许修改
  • 里氏替换：子类可以替换父类出现在父类能够出现的地方
  • 依赖倒置：高层次的模块不依赖低层次的，应该一来一抽象。
  • 接口分离：多个功能用多个接口实现，不要做成一个通用接口

抽象类和接口

抽象类对类找通用性，模板设计；接口对行为进行通用，是行为的规范

相同点

  • 都不能实例化
  • 都位于继承的顶端，用于被继承
  • 都包含抽象方法，子类必须覆写这些方法

不同点

  • 抽象类可以有构造器，接口不能有
  • 抽象类中的修饰符任意，接口默认public
  • 抽象类字段任意，接口字段默认static，final

== 和 equals()

  • ==: 基本数据类型比较值，引用数据类型比较地址是否相等
  • equals(): 如果没有重写，与==一样。否则按照重写的逻辑来
  • String对象因为创建时会从池子中找是否有相同的值，有就给地址，所以用==的话不合适

hashCode() 和 equals()

将对象假如hashSet的时候，如果两个对象hashCode相等，会判断是否equals，如果equals，就不放了。

也就是说，euqals的对象哈希值一定相等，所以改了equals的逻辑，hashcode也要跟着改

IO流

  • InputStream/Reader： 前者是字节流，后者是字符流
  • OutputStream/Writer：同上

BIO,NIO and AIO

  • BIO：Block IO，同步阻塞式IO，就是传统IO，简单使用，并发能力低。一个连接一个线程。 数据的读取写入必须阻塞在一个线程内等待其完成。在活动连接数不是特别高（小于单机1000）的情况下，这种模型是比较不错的，可以让每一个连接专注于自己的 I/O 并且编程模型简单，也不用过多考虑系统的过载、限流等问题
  • NIO：Non IO， 同步非阻塞式IO，客户端和服务器端通过channel通讯，实现了多路复用。一个请求一个线程。
  • AIO：Asynchronous IO，实现了异步非阻塞IO，异步IO操作基于事件和回调机制。一个有效请求一个线程。 它是异步非阻塞的IO模型。异步 IO 是基于事件和回调机制实现的，也就是应用操作之后会直接返回，不会堵塞在那里，当后台处理完成，操作系统会通知相应的线程进行后续的操作。

反射

获取反射的三个方法

  • 建立对象： new 对象 $\rightarrow$ 调用对象.getClass()
  • 通过路径：Class.forName(“package.Class”)
  • 通过类名: ClassName.class

String

String a="a" 和 String a=new String("a")是不一样的，前者放到常量池，后者放到堆内存。

使用hashmap用String做key好处是：String不可变，其hashCode可以缓存下来不需要再次计算

StringBuffer and StringBuilder

StringBuilder 对方法加了同步锁，线程安全；StringBuffer没有加锁，线程不安全。

List

ArrayList and LinkedList 区别

  • ArrayList是动态数组，LinkedList是双向链表
  • 前者随机访问效率高
  • 后者增加和删除的效率高
  • 后者更占内存，因为要存引用
  • 都不保证线程安全

Class.forName 和 classloader

  • 都可以进行类的加载
  • 前者除了将文件加载到jvm，还对类进行解释，执行其中的static
  • 前者可用参数控制是否加载static块

IO

节点流和处理流

  • 从一个特定的IO设备（磁盘、网络等）读写数据，称为节点流
  • 对已存在的流进行连接或封装，通过封装后的流实现数据读写，成为处理流

自定义类加载器

应用场景

  • 加密：编译后的代码加密后无法使用classloader加载，自定义加载器里面先解密，再加载
  • 从非标准的来源加载代码

双亲委派模型过程

  • 从已经加载的类中查询是否已经加载，如有，则返回已经加载的类
  • 如果没有，就让父类加载器加载，采用同样的策略，直到启动类
  • 如果启动类也加载失败了，会抛出异常，并且用当前加载器的findClass加载

Collection框架下面实现比较

  • 实体类实现Comparable接口，并且实现compareTo(T t)方法
  • 创建外部比较器，实现Comparator的compare(T t1, T t2)

Iterator 和 ListIterator区别

  • 使用Iterator可以遍历所有集合，但是只能单一向前方向遍历。不能改变集合中的元素。
  • ListIterator只能遍历List实现的对象，但是可以向前或者向后，同时ListIterator可以在遍历过程中改编List的元素，也可以获取到索引。

快速失败(fail-fast)和安全失败(fail-safe)

快速失败

使用迭代器遍历集合时，如果对内容进行了修改，会抛出错误

原理：迭代过程中使用modCount变量，如果元素发生变化，就会改变modCount值，每次使用next()的时候会检测这个值是否变化，变化了就会抛出异常

场景：java.util的包下面的集合类都是快速失败的，不能多线程修改

安全失败

采用安全失败的集合容器，遍历的时候不是在原本内容上遍历，而是在副本上遍历

原理：迭代时对副本遍历，所以对原集合的修改不能被检测到

缺点：迭代器从遍历开始时拿到副本，此时原集合发生变化不会被感知到

场景：java.util.concurrent包下面的容器都是安全失败，可以多线程并发使用，并发修改

JVM

jvm五大区域：程序计数器、虚拟机栈、本地方法栈、java堆、方法区

  • 程序计数器：线程私有，很小的内存空间，作为当前线程的行号指示器，记录当前虚拟机正在执行的线程指令地址
  • 虚拟机栈：线程私有，每个方法执行的时候都会创建一个栈帧，用于存储局部变量表、操作数、动态链接和方法返回等信息，线程请求栈深度超过虚拟机允许的最大深度时会抛出溢栈错误
  • 本地方法栈：线程私有，保存native方法的信息，jvm创建的线程调用native方法后，jvm不会创建栈帧，而是简单地动态链接并直接调用该方法
  • 堆：所有线程共享的一块内存，所有对象的实例和数组都在堆上分配内存，垃圾回收常在这个区域发生
  • 方法去：存放已经被加载的类信息、常量、静态变量、即时编译器编译后的代码。jdk1.8后不存在了，被元数据区替代，方法区被分为两个部分：加载的类信息和运行时常量池。加载的类信息保存在元数据区中，运行时常量保存在堆中。

heap和stack的区别

申请方式

  • stack： 系统自动分配，声明的局部变量系统会自动在栈中分配空间
  • heap：需要手动申请，例如new

申请后的系统响应

  • stack： 只要栈的剩余空间足够，系统就会提供空间，否则报异常
  • heap： 操作系统有一个记录空闲内存地址的链表，系统收到程序申请的时候，会遍历该链表，找到第一个符合申请空间的堆节点，将节点从内存中去掉，把空间分配给程序。由于找到的空间可能偏大，系统会把多余的空间收回放入空闲链表

申请大小的限制

  • stack： 栈是向低地址扩展的数据结构，是连续的存储空间。栈顶地址和栈的最大容量是系统预设好的，因此如果申请的空间朝服哦栈的剩余空间，会报错，获得的空间较小
  • heap： 堆是向高地址扩展的数据结构，不连续。堆的大小受限制于计算机系统的虚拟内存。获取堆空间灵活性较大，空间也较大

java中的垃圾回收算法

标记清除法

  • 第一步： 利用可达性去遍历内存，把存活对象和垃圾对象标记
  • 第二步： 再遍历一遍，将所有标记的对象回收掉
  • 特点： 效率不高，标记和清除后会产生大量不连续空间。

标记整理法

  • 第一步： 利用可达性去遍历内存，把存活对象和垃圾对象标记
  • 第二步： 将所有存活对象向一端移动，端边界的对象都回收
  • 特点： 适用于存活对象多，垃圾少的情况

复制算法

将内存按照容量大小分为相等两块，每块只用一次，一块用完后，将存活对象移到另一块，再把使用过的内存空间清除

特点：不产生空间碎片，内存使用率极低

分代收集算法

根据对象存活周期不同，将内存划分为几块。java虚拟机将内存分为新生代和老生代，新生代中大量对象死去和少量对象存活，使用复制算法；老生代中对象存活率高，采用标记清理或者标记整理。
